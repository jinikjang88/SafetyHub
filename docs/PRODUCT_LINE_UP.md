## 1. SafetyKit (산업 설비용 안전 장치)

### 개념

기존 설비를 교체하지 않고 **전원선에 끼워넣는 방식**으로 안전 시스템을 추가합니다.

**설치 방식:**

- 기존: 콘센트 → 설비장비
- SafetyKit 적용: 콘센트 → SafetyKit → 설비장비
- 설치 시간: 30초 (플러그 방식)

### 제품 구성

| 구성요소 | 설명 |
| --- | --- |
| INPUT 포트 | 전원 입력 |
| OUTPUT 포트 | 설비 연결 |
| SSR 릴레이 | 40A 무접점 전원 차단 |
| MCU (ESP32) | 센서 입력, 네트워크 통신, 로컬 판단 |
| 긴급정지 버튼 | 물리적 비상 정지 |
| 상태 LED | 🟢 정상 / 🟡 경고 / 🔴 차단중 |

### 제품 라인업

| 모델 | 사양 | 가격 | 타겟 용도 |
| --- | --- | --- | --- |
| **SafetyKit Lite** | 단상 220V/16A, WiFi | 8만원 | 소형 기계, 공작실 |
| **SafetyKit Pro** | 삼상 380V/32A, WiFi+LoRa+4G | 25만원 | 공장 설비 |
| **SafetyKit Industrial** | 삼상 380V/63A+, 이중화 통신 | 50만원 | 대형 프레스, 컨베이어 |

### 내부 회로 구성

| 컴포넌트 | 역할 |
| --- | --- |
| SSR (Solid State Relay) | 40A 무접점 릴레이, 전원 차단 실행 |
| ESP32 MCU | 센서 데이터 수집, 통신, 로컬 판단 |
| 모션 센서 (PIR) | 작업자 접근 감지 |
| 거리 센서 (초음파) | 안전 거리 측정 |
| 전류 센서 (CT) | 설비 작동 상태 모니터링 |

## 2. LifeGuard Band (작업자용 생체 모니터링)

### 핵심 원리: Pulse Transit Time (PTT)

두 지점(팔, 다리)에서 생체 신호를 측정하여 **맥파 전달 시간**을 분석합니다.

| 상태 | 심장 → 팔 | 팔 → 다리 | 판정 |
| --- | --- | --- | --- |
| 정상 | 0.1초 | 0.15초 | 일정한 간격 |
| 이상 | 0.1초 | 0.4초 | ⚠️ 지연 - 혈류 문제 의심 |

### 감지 가능한 이상 징후

| 징후 | 감지 방식 |
| --- | --- |
| 혈전 (DVT) | 다리 신호 지연/약화 |
| 말초동맥질환 | 신호 전달 이상 |
| 부정맥 | 두 지점 리듬 불일치 |
| 혈압 급변 | PTT 변화로 추정 |
| 쇼크/실신 전조 | 말초 혈류 감소 |

### 제품 구성

| 구성품 | 설명 |
| --- | --- |
| **Band A (팔)** | PPG 센서, 가속도계 (낙상 감지), 온도 센서, SOS 버튼 |
| **Band B (다리)** | PPG 센서, 가속도계 (움직임), 온도 센서 |
| **스마트폰 App** | 실시간 모니터링, AI 이상 감지, 긴급 신고 연동 |

### 긴급 신고 플로우

| 단계 | 동작 | 조건 |
| --- | --- | --- |
| 1. 이상 감지 | AI 분석 | Band 이상 신호 |
| 2. 1차 확인 | 10초 카운트다운 + "괜찮으세요?" 알림 | 위험 판정 |
| 3. 사용자 응답 | "괜찮아요" → 경고 해제 (기록 저장) | 응답 시 |
| 4. 무응답 | 🚨 119 자동 신고 + GPS 위치 전송 | 응답 없음 |
| 5. 보호자 알림 | 위치 공유 | 신고 완료 |

### 타겟 사용자

| 대상 | 니즈 | 가치 |
| --- | --- | --- |
| 👴 독거 노인 | 혼자 쓰러져도 발견 안 됨 | 자동 감지 + 신고 |
| 🏭 산업 현장 근로자 | 고온/위험 환경 작업 | 열사병, 실신 조기 감지 |
| 🏃 마라톤/등산객 | 무리한 운동 중 위험 | 심장마비 조기 경고 |
| ❤️ 심장/뇌졸중 고위험군 | 24시간 모니터링 필요 | 골든타임 확보 |

## 3. Robot Control Simulator

### 목적

실제 로봇 없이 **수만 대 규모의 가상 로봇**을 시뮬레이션하여 관제 시스템을 검증합니다.

| 항목 | 설명 |
| --- | --- |
| 대규모 부하 테스트 | 실제 로봇 구매 없이 10,000+ 로봇 시뮬레이션 |
| 카오스 엔지니어링 | 다양한 장애 시나리오 검증 |
| 알고리즘 최적화 | 분배 알고리즘 튜닝 |
| 클린 아키텍처 강제 | 시뮬레이터와 실제 로봇이 같은 인터페이스 |

### 카오스 시나리오

| 시나리오 | 설명 | 검증 항목 |
| --- | --- | --- |
| MASS_FAILURE | 30% 로봇 동시 고장 | 장애 복구, 작업 재분배 |
| NETWORK_PARTITION | 특정 구역 통신 두절 | 로컬 자율 동작 |
| TRAFFIC_SPIKE | 대량 작업 동시 생성 | 큐 처리량, 응답시간 |
| CHARGING_STATION_FAILURE | 충전소 고장 | 대체 경로 탐색 |
